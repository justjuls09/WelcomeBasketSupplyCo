# Sprint 14 – Agentic Usability Audit

## Persona Gallery

### Maya – Busy Mobile User
Goal: Order a welcome kit quickly on her phone during a short break  
Pain Points: Gets frustrated when buttons are unclear and layout is confusing  

### Alex – Beginner User
Goal: Complete an order correctly without mistakes  
Pain Points: Feels overwhelmed when there is no clear starting point or instructions  

### Jordan – Accessibility User
Goal: Easily read and navigate the app  
Pain Points: Struggles with unclear labels, small text, and poor visual structure  

### Chris – Time‑Pressed Professional
Goal: Complete order in under 2 minutes  
Pain Points: Too many steps and no progress indicator  

---

## Testing Results

"I just want to order this welcome kit quickly on my phone during my lunch break..."

### Confusion Points
- Button labels unclear (Submit/Proceed not descriptive)
- Hard to know where to start
- Important info hidden in dropdowns
- Technical language confusing
- Error messages unclear

### Frustrations
- Mobile layout difficult to use
- Too many steps
- No progress indication
- No clear confirmation page
- Losing work is frustrating

---

## Top 3 Issues
1. Unclear button labels
2. No clear starting point for users
3. Poor mobile usability

## Changes Made
- Renamed buttons (e.g., "Submit" → "Submit Order") to make actions clearer  
- Added instructions at the top to guide users on where to start  
- Improved layout structure by adding section labels to reduce confusion  

---

## Reflection

1. I thought my layout was clear, but users did not know where to start, which showed me my design was not as intuitive as I assumed.

2. Streamlit made me think more about visual layout and user flow compared to console apps, where usability issues are less visible.

3. The agent identified clarity and usability issues that I did not initially notice, especially around instructions and button labels.
